# Redfish ISO Boot Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move OVE node ISO boot to the bastion — site.yml provisions a self-contained boot playbook and static Redfish inventory onto the bastion, run locally with `ansible-playbook /root/boot-ove-nodes.yml -i /root/ove-redfish-inventory.ini`.

**Architecture:** The `bastion_configure` role gains a `boot_prep.yml` task file that installs ansible-core, copies the ISO to the web root, and templates a static Redfish inventory and boot playbook onto the bastion. Node names (not UUIDs) are used as Redfish resource IDs — sushy-emulator redirects name-based URLs to UUIDs.

**Tech Stack:** `ansible.builtin.dnf`, `ansible.builtin.command` (ansible-galaxy), `ansible.builtin.copy`, `ansible.builtin.template`, `community.general.redfish_command`

### Task 4: Create boot playbook template

**Files:**
- Create: `roles/bastion_configure/templates/boot-ove-nodes.yml.j2`

- [ ] **Step 1: Create the template**

Static values (`sushy_port`, `sushy_username`, `sushy_password`, ISO filename) are rendered at template time. `inventory_hostname` is preserved using `{% raw %}` so it evaluates when the generated playbook runs on the bastion.

```jinja2
---
# boot-ove-nodes.yml — Boot OVE nodes from ISO via Redfish
# Generated by site.yml — run from this host:
#   ansible-playbook /root/boot-ove-nodes.yml -i /root/ove-redfish-inventory.ini

- name: Boot OVE nodes from ISO via Redfish
  hosts: ove_nodes
  connection: local
  gather_facts: false
  tasks:
    - name: Insert ISO as virtual media
      community.general.redfish_command:
        category: Manager
        command: VirtualMediaInsert
        baseuri: "localhost:{{ sushy_port }}"
        username: "{{ sushy_username }}"
        password: "{{ sushy_password }}"
        virtual_media:
          image_url: "http://localhost/{{ ove_agent_iso_path | basename }}"
          media_types:
            - CD
        resource_id: "{% raw %}{{ inventory_hostname }}{% endraw %}"
        no_log: true

    - name: Set one-time boot from CD
      community.general.redfish_command:
        category: Systems
        command: SetOneTimeBoot
        baseuri: "localhost:{{ sushy_port }}"
        username: "{{ sushy_username }}"
        password: "{{ sushy_password }}"
        bootdevice: Cd
        resource_id: "{% raw %}{{ inventory_hostname }}{% endraw %}"
        no_log: true

    - name: Power on
      community.general.redfish_command:
        category: Systems
        command: PowerOn
        baseuri: "localhost:{{ sushy_port }}"
        username: "{{ sushy_username }}"
        password: "{{ sushy_password }}"
        resource_id: "{% raw %}{{ inventory_hostname }}{% endraw %}"
        no_log: true
```

- [ ] **Step 2: Commit**

```bash
git add roles/bastion_configure/templates/boot-ove-nodes.yml.j2
git commit -m "feat: add boot playbook template"
```

### Task 5: Create boot_prep.yml and wire into main.yml

**Files:**
- Create: `roles/bastion_configure/tasks/boot_prep.yml`
- Modify: `roles/bastion_configure/tasks/main.yml`

- [ ] **Step 1: Create `roles/bastion_configure/tasks/boot_prep.yml`**

```yaml
---
- name: Install ansible-core
  ansible.builtin.dnf:
    name: ansible-core
    state: present

- name: Install community.general Ansible collection
  ansible.builtin.command:
    cmd: ansible-galaxy collection install community.general
  changed_when: false

- name: Copy OVE agent ISO to web root
  ansible.builtin.copy:
    src: "{{ ove_agent_iso_path }}"
    dest: "/var/www/html/{{ ove_agent_iso_path | basename }}"
    mode: "0644"

- name: Template Redfish inventory
  ansible.builtin.template:
    src: ove-redfish-inventory.ini.j2
    dest: /root/ove-redfish-inventory.ini
    mode: "0644"

- name: Template boot playbook
  ansible.builtin.template:
    src: boot-ove-nodes.yml.j2
    dest: /root/boot-ove-nodes.yml
    mode: "0644"
```

- [ ] **Step 2: Add boot_prep include to `roles/bastion_configure/tasks/main.yml`**
